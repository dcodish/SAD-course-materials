# חומרי הקורס — לפי שלבי ה-SDLC

החומרים מאורגנים לפי **שלבי מחזור חיי הפיתוח (SDLC)**, כדי שתוכלו לעקוב אחרי הדוגמה הרצה של הקורס — **מערכת הזמנת אוכל בקמפוס** — שלב אחרי שלב, בדיוק כפי שמתקדמים בהרצאות.

בכל שלב יש **שתי גרסאות**: *חומר גלם* (מה שהאנליסט/ית אספ/ה — לא מאומת, מלא סתירות ורעש; להדגמה בכיתה למה אי אפשר לסמוך על הנתונים כמו שהם) ו*תוצר מאומת* (התוצר הסופי אחרי הניתוח — דוגמת ייחוס לפרויקט שלכם).

- **[1 — ייזום](1-ייזום/)** — מזהים בעיה עסקית ומגדירים מטרה · הרצאה 1
- **[2 — חקר מצב קיים](2-חקר-מצב-קיים/)** — ממפים את התהליך הקיים ובעלי העניין · הרצאות 1–2
- **[3 — ניתוח דרישות](3-ניתוח-דרישות/)** — מנסחים מה המערכת צריכה לעשות · הרצאות 3, 4–5
- **[4 — חקר ישימות](4-חקר-ישימות/)** — בודקים אם כדאי ואפשרי לבנות · הרצאה 10
- **[5 — עיצוב](5-עיצוב/)** — מתכננים את מבנה המערכת והתנהגותה · הרצאות 6–7, 8, 9
- **[6 — פיתוח](6-פיתוח/)** — כותבים את הקוד (היום — הרבה עם AI) · הרצאה 13
- **[7 — בדיקות](7-בדיקות/)** — מוודאים שהמערכת עושה את הנדרש
- **[8 — הטמעה ותפעול](8-הטמעה-ותפעול/)** — מעלים לאוויר, מתחזקים ומשפרים

כל תיקייה מתמלאת ככל שמתקדמים בקורס.
